# 演讲培训技能

## Overview

本 skill 辅助演讲者完成演讲/培训材料的全流程制作：从主题大纲碰撞、名人名言筛选、演示脚本编写、完整逐字稿，到 HTML 格式 PPT 生成。

## 核心产出物

| 产出物 | 文件命名规范 | 说明 |
|--------|-------------|------|
| 演讲逐字稿 | `{日期}培训素材/演讲逐字稿.md` | 含章节结构的完整演讲内容 |
| 演示指令脚本 | `{日期}培训素材/演示指令脚本.md` | 现场演示的分步指令 |
| HTML PPT | `{标题}.html` 或 `{标题}-嵌入版.html` | 独立运行的幻灯片 |

## Workflow

### Step 1: 主题信息收集

使用 `AskUserQuestion` 工具向用户提问。**注意**：`AskUserQuestion` 单次最多支持4个问题，7个信息需分两批提问。

**第一批（4个问题）**：
1. **演讲主题** — 具体要讲什么？
2. **目标受众** — 业务/HR/技术？人数规模？
3. **时长** — 15min / 30min / 45min？
4. **场合** — 园区分享 / 内部培训 / 大会演讲 / 其他？

**第二批（3个问题）**：
5. **核心信息** — 用户想传递的 2-3 个关键观点
6. **参考资料** — 是否有现成的文档、图片、素材？（如有请用户提供）
7. **演讲者信息** — 姓名、职位、公司（用于封面）

### Step 2: 名人名言筛选（如需要）

根据主题，筛选 AI 领域前沿人士的最新观点（2025-2026年）：

**常用来源：**
- Dario Amodei (Anthropic CEO) — AI risk essays, 2026 predictions
- Sam Altman (OpenAI CEO) — AI agents, AI Ascend conference
- Jimmy Ba (xAI 联合创始人) — 2026 年离职 X 后发言
- Ethan Mollick (沃顿商学院) — AI 应用实操观察
- Geoffrey Hinton、Yoshua Bengio — AI 教父级别

**筛选标准：**
- 有名有姓，禁止模糊引用
- 时间新鲜（2025-2026年优先）
- 与演讲主题相关
- 每位名人 2 句金句

### Step 3: 生成演讲逐字稿

**结构模板（通用版）：**

```
## 第一章：观点引入（~1分钟）
- 用一个反常识判断或数据开场
- 引用 1-2 位名人的话
- 提出核心矛盾/K型分化

## 第二章：是什么（~2-3分钟）
- 用比喻解释核心概念
- 旧模式 vs 新模式对比
- 包含演示环节（可嵌入PPT页码引用）

## 第三章：实战场景（~3-4分钟）
- 2-3 个真实使用案例
- 每个案例：背景→操作→结果
- 可引用用户提供的截图

## 第四章：为什么推荐（~2-3分钟）
- 平台对比（如果涉及工具选型）
- 三层优势拆解
- 最新动态/官方插件

## 第五章：未来期待（~1-2分钟）
- 短期可实现的能力
- 个人/企业落地建议

## 收尾（~2分钟）
- 回扣开场观点
- 行动号召
- 结束语
```

**写作风格：**
- 口语化、接地气，像在跟观众聊天
- 用"我觉得/说实话/兄弟"这类口语连接词拉近距离
- 每个章节有具体的故事和数据支撑
- 禁止空洞的"赋能"类词汇，少用官话套话
- 名人名言要标注：姓名、职位、出处、时间
- 多用短句，少用复杂从句，演讲时容易卡壳

### Step 4: 生成演示指令脚本

**脚本结构：**

```
### 演示N：{演示名称}

**目标**：让 AI 帮你完成什么具体任务

**指令示例**：
```
粘贴给 AI 的具体指令
```

**操作步骤**：
1. 第一步
2. 第二步
3. 第三步

**风险提示**：
- 权限确认
- 备选 URL
- 失败处理
```

**常用演示类型：**
- AI 读文档 → 总结 → 飞书云文档
- AI 读网页 → 总结 → 飞书云文档
- AI 创建飞书日程
- AI 控制电脑执行命令

### Step 5: HTML PPT 生成（调用 frontend-slides skill）

**前置确认：**
1. 确认用户有无可直接使用的图片素材
2. 了解投影环境（亮度偏好）

**风格选择流程：**
1. 生成 3 个风格预览（见 frontend-slides skill Phase 2）
2. 用户选择后生成完整 PPT
3. 如用户要求嵌入图片，使用 Python base64 转换

**PPT 结构建议（15分钟演讲）：**

| 页码 | 内容 |
|------|------|
| 1 | 封面（标题 + 演讲者信息 + 小红书/社交卡片） |
| 2 | 三位预言家（名人名言页） |
| 3 | 核心转变（旧模式 vs 新模式） |
| 4 | 概念解释（是什么） |
| 5 | 演示一（高亮背景） |
| 6 | 工具介绍（AI助手） |
| 7 | 实战场景截图 |
| 8 | 自动化案例 |
| 9 | 全链路案例 |
| 10 | 平台选择理由 |
| 11 | 工具能力表 |
| 12 | 未来期待 |
| 13 | 企业培训案例 |
| 14 | 收尾语 |
| 15 | 感谢页 |

**图片嵌入处理（如需要）：**

```python
import base64, os

folder = "/path/to/素材文件夹/"
html_file = "PPT.html"
images = {
    "头像.jpg": "image/jpeg",
    "截图.png": "image/png",
}

with open(os.path.join(folder, html_file), "r", encoding="utf-8") as f:
    html = f.read()

for filename, mime in images.items():
    with open(os.path.join(folder, filename), "rb") as f:
        b64 = base64.b64encode(f.read()).decode("ascii")
    html = html.replace(f'src="{filename}"', f'src="data:{mime};base64,{b64}"')

with open(os.path.join(folder, html_file.replace(".html", "-嵌入版.html")), "w", encoding="utf-8") as f:
    f.write(html)
```

## 文件输出规范

### 演讲逐字稿

路径：`{日期}培训素材/演讲逐字稿.md`

结构：
- 文件头（演讲标题、日期、演讲者）
- 素材索引表（章节→对应图片）
- 正文（分章节，含 Timing 预估）

### 演示指令脚本

路径：`{日期}培训素材/演示指令脚本.md`

结构：
- 概述（几个演示，核心目标）
- 每个演示的完整脚本
- 演示前检查清单

### HTML PPT

路径：`{标题}.html`（外部引用版）
路径：`{标题}-嵌入版.html`（全嵌入版，发送给其他人时使用）

命名示例：`演讲主题-日期.html`

## 素材收集清单

开始制作前，确认以下素材已就位：

- [ ] 演讲者头像（用于封面）
- [ ] 名人头像（如有 PPT 第2页需求）
- [ ] 实战场景截图（AI助手对话、自动化成果等）
- [ ] 产品/工具截图（CLI 技能表等）
- [ ] 企业培训相关截图（如有）

## 注意事项

1. **名人名言必须核查**：引用的数据、职位、时间必须在 2025-2026 年范围内
2. **面向非技术受众**：解释 AI 概念时用比喻，不用术语
3. **投影环境**：优先选择亮色系风格（warm cream / light background），避免纯黑深色背景
4. **图片命名**：用有意义的命名，避免中文长文件名（部分系统兼容问题）
5. **HTML 嵌入**：发送给其他人时使用嵌入版，避免图片路径丢失
